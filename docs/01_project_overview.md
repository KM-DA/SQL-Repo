# SQL A/B Test & Cohort Analysis

## 1. 프로젝트 개요
본 프로젝트는 서비스 내 기능/정책 변경이
사용자 행동(KPI)에 미치는 영향을
SQL 기반으로 분석하는 것을 목표로 한다.

### 실험 상황 (A/B Test 시나리오)
- 신규 기능(또는 UI/정책) 적용 여부에 따른 성과 비교
- 동일 기간, 동일 유입 채널 기준으로 실험 진행

### 가설 (Hypothesis)
- H0(귀무가설): 실험군과 대조군의 KPI 차이는 없다.
- H1(대립가설): 실험군의 KPI가 대조군보다 유의미하게 높다.

### 실험 목표 및 KPI
- 1차 KPI: 전환율(Conversion Rate)
- 2차 KPI: CTR, 구매율
- 보조 지표: 사용자 유지율, 평균 행동 수

---

## 2. 데이터 구조 설명
본 프로젝트에서는 실제 사내 데이터가 아닌
**Synthetic Data(가상 데이터)** 를 사용한다.

### 주요 테이블
- users
- events
- transactions

### 컬럼 정의 (요약)
- user_id: 사용자 고유 ID
- group_flag: A/B 그룹 구분 (A=control, B=treatment)
- event_date: 이벤트 발생 날짜
- event_type: 클릭/구매 등 이벤트 유형
- revenue: 구매 금액

---

## 3. A/B 실험 설계
### 실험군 / 대조군 정의
- Control Group (A): 기존 정책 사용자
- Treatment Group (B): 신규 정책 사용자

### 샘플 사이즈 개념
- 통계적 검정을 위한 최소 표본 수 개념만 언급
- 실제 계산은 범위 추정 수준으로 제한

### 실험 기간
- 동일 기간 내 비교 (예: 4주)

---

## 4. 분석 접근 방식 (SQL)
- 실험군/대조군 데이터 추출
- KPI 집계 (CTR, CVR, 구매율)
- 그룹별 평균/분산/샘플 수 계산
- 통계 검정용 집계 테이블 생성
- 기간별 KPI 변화 추적
- 이상치 및 비정상 사용자 필터링

---

## 5. 통계적 유의성 검정
- T-test
- Chi-square test
- p-value 해석
- 효과 크기(effect size) 비교

---

## 6. 결과 해석 및 인사이트
- KPI 변화 요약
- 실험 성공/실패 판단
- 비즈니스 관점의 의미 해석

---

## 7. 결론 및 개선 제안
- 실험 결과 요약
- 다음 실험에 대한 개선 아이디어
- 추가 분석 방향
